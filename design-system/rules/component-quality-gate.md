# بوّابة جودة المكوّن — Component Quality Gate

> **القرار:** البروتوكول لا يقول «ابنِ Table» فقط. أي مكوّن لا يُعتبر منجَزًا حتى يمرّ هذه البوّابة: (أ) المعايير العامة · (ب) معايير المكوّن الخاصّة · (ج) قاعدة الاختيار الصحيح.
> **مكانها:** بوّابة ثالثة بجانب [Design System Gate](../../ROLES.md) و[Accessibility Gate](../../ROLES.md). تُفحَص لكل مكوّن أثناء البناء/المراجعة. **مكوّن يفشلها = العمل غير منجَز.**
> القيم/الحالات تفصيلها في [components/](../components/)؛ هذا الملف يجعلها **pass/fail مُلزِمة**.

---

## 1. المعايير العامة (كل مكوّن تفاعلي/بيانات)

- **كل الحالات المطلوبة موجودة:** default · hover · focus-visible · active · disabled · loading (حسب الانطباق) — ولمكوّنات البيانات: empty · no-results · error.
- **الكيبورد والتركيز** كاملان (← [Accessibility Gate](../../design-protocols/06-accessibility.md)).
- **RTL منطقي** (خصائص `inline-*`/`start/end`، لا اتجاهي) (← [04](../../design-protocols/04-arabic-rtl-ux.md)).
- **توكنز دلالية فقط** عبر عقد الربط (← [component-tokens](../foundations/component-tokens.md) · [Design System Gate]).
- **إجراء أساسي واحد** في السياق؛ لا خلط primary بثانوي.
- **لا ازدحام، لا حدود بلا داعٍ، لا زخرفة بلا غرض** (← [Visual & Experience / Output stance](../../ROLES.md)).
- **مستجيب** عبر نقاط الكسر.
- **يحقّق معايير القبول البصري** في §10 من spec المكوّن.

## 2. معايير إلزامية لكل مكوّن (أمثلة محورية)

### Table — لا يُعتبر منجَزًا إلا بـ
رأس واضح · كثافة صفّ مناسبة · hover · محدّد (إن وُجد) · empty · loading · error · no-results · ترقيم · بحث/فلاتر (إن لزم) · سلوك مستجيب (تمرير أفقي أو بطاقة-لكل-صفّ، **لا قصّ صامت**) · محاذاة RTL (أرقام `end`+`tabular-nums`) · إجراءات واضحة · **غير مزدحم** · لا حدود كثيرة بلا داعٍ · لا خلط primary/secondary في الإجراءات · أحجام أعمدة منضبطة (لا تمدّد عشوائي). (← [table.md](../components/table.md))

### Modal — لا يُعتبر منجَزًا إلا بـ
عنوان واضح يصف القرار · وصف مختصر للأثر · **زر أساسي واحد** + إلغاء/تراجع · focus trap + Esc · إغلاق آمن (لا يفقد إدخالًا دون تحذير) · padding مضبوط (`--space-lg`) · عرض مناسب (sm/md/lg، لا أوسع) · RTL (العنوان `inline-start`، ✕ `inline-end`) · **✕ غير عشوائي/قريب من النصّ**. **لا يُستخدم** لمحتوى طويل ولا لعملية متعدّدة الخطوات → Drawer/Page؛ **لا مودال يفتح مودالًا**. (← [modal.md](../components/modal.md))

### Form — لا يُعتبر منجَزًا إلا بـ
**تسميات ظاهرة دائمًا** (لا الاعتماد على placeholder) · رسائل خطأ **تحت الحقل** · required/optional واضح (نهج واحد لا خلط) · grouping منطقي (`fieldset`) · حالة submit · حالة success · validation (عند blur/submit لا كل حرف) · مسافات ثابتة (لا قفز تخطيط) · RTL · **تقسيم النماذج الطويلة** (لا حقول كثيرة بلا أقسام). (← [form.md](../components/form.md))

### Button / Input (مختصر)
- **Button:** `<button>` لا `<div>` · primary واحد · loading يحفظ العرض · focus ring بديل (لا إزالة `outline` بلا بديل) · نصّ فعل. (← [button.md](../components/button.md))
- **Input:** تسمية مرتبطة `<label for>` · حالات الحدّ (default/hover/focus/error/disabled/readonly) · خطأ `aria-invalid`+`aria-describedby` · bidi للبريد/URL/الأرقام. (← [input.md](../components/input.md))

> بقية المكوّنات: المعايير العامة §1 + قسمَا «الحالات» و«معايير القبول البصري» في spec كلٍّ.

## 3. قاعدة الاختيار الصحيح (المكوّن المناسب للمهمّة)

البوّابة ترفض المكوّن **الخطأ** حتى لو نُفِّذ جيدًا:

| الحاجة | المناسب | ليس |
|---|---|---|
| قرار/إدخال **قصير مركّز** | **Modal** | Drawer/Page |
| محتوى/سياق **جانبي أو أطول** | **Drawer** | Modal |
| عملية **متعدّدة الخطوات** أو محتوى طويل | **Page / Wizard / Stepper** | Modal |
| إشعار **غير معطِّل** | **Toast** | Modal |
| سجلّات **متجانسة** تُقارَن/تُفرَز | **Table** | Cards/List |
| عنصر واحد أو بنية **غير متجانسة** | **Card / تفصيل** | Table |
| قائمة قصيرة بلا مقارنة أعمدة | **List** | Table |

## 4. الإنفاذ (متى وكيف)

- **متى:** أثناء البناء (لكل مكوّن) وفي المراجعة. جزء من مرور الأدوار ([ROLES](../../ROLES.md)).
- **المُلّاك:** **UX & IA** (الاكتمال + الاختيار الصحيح) · **Visual & Experience** (لا ازدحام/الجودة) · **Design System** (عقد الربط + التوكنز).
- **الفشل يحجب:** مكوّن لا يحقّق §1 + معاييره الخاصّة + قاعدة الاختيار = **البوّابة لا تمرّ**، والعمل غير منجَز ([07](../../design-protocols/07-definition-of-done.md)).
- **القيد:** البوّابة معايير قبول موثّقة (تُفحَص يدويًا/بالمراجعة)، لا أداة آلية — اتساقًا مع قيد لا-أدوات.
