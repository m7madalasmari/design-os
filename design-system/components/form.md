# النموذج (طبقة التركيب) — Form Composition

> هذا الملف **ليس مكوّنًا ذرّيًا** بل طبقة تركيب الحقول في نموذج: التخطيط، توقيت التحقّق، موضع الأزرار.
> الحقول الذرّية لها ملفّاتها: [input](input.md) · [select](select.md) · [checkbox](checkbox.md).
> القيم من [tokens.md](../foundations/tokens.md). المسافات من [spacing.md](../foundations/spacing.md).

## الغرض
جمع إدخال المستخدم بأقلّ احتكاك وأقلّ خطأ. الوضوح أهمّ من الجمال هنا تمامًا ([design-principles](../foundations/design-principles.md) §1).

## التشريح (الحقل الواحد)
```
التسمية *            ← دائمًا ظاهرة، فوق الحقل، --font-weight-medium
[ الحقل           ]  ← ارتفاع --size-control-md، حدّ --color-border-default
نص مساعدة            ← --font-size-sm، --color-text-tertiary (اختياري)
رسالة الخطأ          ← --color-danger-fg، تظهر عند الخطأ فقط
```
المسافات تتبع القرب ([spacing](../foundations/spacing.md)): تسمية↔حقل `--space-xs`، حقل↔حقل تالٍ `--space-md`.

## القواعد الحاكمة
- **التسمية دائمًا ظاهرة فوق الحقل.** ممنوع استخدام `placeholder` بديلًا عن التسمية ([forbidden-patterns](../rules/forbidden-patterns.md)): النائب يختفي عند الكتابة فيضيع السياق ويفشل الوصول.
- الـ`placeholder` (إن وُجد) **مثال تنسيق** لا تعليمات: «مثال: 0512345678».
- علّم الحقول: المطلوب بـ`*` + `aria-required`، أو علّم الاختياري بـ«(اختياري)» — **ثبّت أحد النهجين** ولا تخلط.
- نصّ المساعدة قبل الخطأ، مكانه ثابت تحت الحقل (لا قفز تخطيط عند ظهور الخطأ).
- مجموعة الحقول المترابطة داخل `<fieldset>` بـ`<legend>`.

## الأنواع
نصّ، نصّ طويل (textarea)، رقم، بريد، هاتف، تاريخ، قائمة منسدلة (select)، خانة اختيار (checkbox)، زر اختيار (radio)، مفتاح (toggle)، رفع ملف. كلها تشترك في التوكنز والحالات أدناه.

## الحالات (إلزامية)
| الحالة | المظهر |
|---|---|
| `default` | حدّ `--color-border-default` |
| `hover` | حدّ `--color-border-strong` |
| `focus` | حدّ `--color-border-focus` + `--shadow-focus` |
| `filled` | كما default مع نصّ `--color-text-primary` |
| `error` | حدّ `--color-danger-border` + رسالة `--color-danger-fg` + `aria-invalid` |
| `disabled` | `--color-bg-disabled`، `--color-text-disabled`، `aria-disabled` |
| `readonly` | بلا حدّ تفاعلي، نصّ منسوخ |

## التحقّق (validation)
- وقّت التحقّق **عند مغادرة الحقل (blur)** أو عند الإرسال — لا عند كل ضغطة (يُربك).
- اعرض الخطأ بجانب الحقل المعني، وملخّصًا أعلى النموذج إن كثرت الأخطاء.
- صياغة الخطأ تصف الشرط الصحيح لا اللوم ([ux-writing §الأخطاء](../foundations/ux-writing.md)).
- لا تُعطّل زر الإرسال بصمت؛ إن عُطّل، وضّح السبب.

## دورة الإرسال (submit lifecycle — إلزامية)
- **أثناء الإرسال:** زر الإرسال `loading` (عرض ثابت) + `aria-busy="true"` على النموذج + منع الإرسال المزدوج. لا تجمّد الواجهة بصمت.
- **نجاح:** تأكيد صريح — toast/alert نجاح، ووضوح ما حُفِظ/تغيّر وما التالي (إفراغ النموذج أو الانتقال). لا «نجاح صامت».
- **خطأ خادم:** ملخّص خطأ أعلى النموذج (`role="alert"`) + **إعادة التركيز لأول حقل خاطئ**؛ تُحفَظ مدخلات المستخدم (لا تُمسَح). أخطاء الحقول بجانبها.
- **استرداد:** يمكن للمستخدم التصحيح وإعادة الإرسال دون فقد بيانات.

## RTL و bidi
- الحقول العربية `dir` تتبع الجذر (rtl)، المؤشّر يبدأ يمينًا.
- حقول البريد/URL/الأرقام: `dir="ltr"` على الحقل، مع بقاء التسمية والخطأ والتخطيط RTL ([rtl-guidelines §5](../foundations/rtl-guidelines.md)).
- علامة المطلوب وأيقونة الحالة في الجهة المنطقية، بخصائص `inline-*`.
- الأرقام لاتينية ([typography](../foundations/typography.md)).

## أزرار النموذج
- إجراء أساسي واحد («احفظ») + ثانوي («تراجع»). موضع ثابت (أسفل النموذج)، نمط ترتيب موحّد عبر التطبيق.
- في نموذج طويل: شريط إجراءات لاصق أسفل المحتوى.

## الوصول
- لكل حقل `<label for>` مرتبط (أو `aria-labelledby`).
- الخطأ مرتبط بـ`aria-describedby`؛ يُعلَن لقارئ الشاشة.
- ترتيب تركيز يتبع القراءة RTL؛ تشغيل كامل بلوحة المفاتيح.
- هدف لمس 44px لخانات الاختيار وأزرار الاختيار (وسّع منطقة النقر للتسمية).

## ممنوع
- `placeholder` بدل التسمية.
- إخفاء التسميات «لأناقة» التصميم.
- تحقّق فوري مزعج عند كل حرف.
- قفز التخطيط عند ظهور الخطأ.
- توزيع حقول مترابطة على أعمدة لا يستدعيها المعنى.
- خلط نظامَي تعليم المطلوب/الاختياري.
