# الحالة الفارغة — Empty State

> القيم من [tokens.md](../foundations/tokens.md). تطبيق المبدأ 6 (حالات مكتملة).
> **قاعدة بنية إلزامية:** عنوان الحالة الفارغة عنصر **عنوان حقيقي** (`<h1>`–`<h3>`)، لا `<p>`/`<div>` — انظر §«بنية العنوان والوصول». (أُضيفت لمنع تكرار خطأ كسر شجرة العناوين.)
> **سياسة الرسوم (المرجع النهائي):** هذا المكوّن يحكم استخدام الأيقونة/الرسم في الحالات الفارغة. عند أي تعارض مع [05](../../design-protocols/05-visual-polish-and-motion.md) **يفوز المكوّن (Component spec wins)**.

## 1. متى يُستخدم
حين لا يوجد محتوى لعرضه: أول استخدام، لا نتائج بعد فلترة، تعذّر جلب البيانات، أو لا صلاحية. تشرح **السبب** وتعرض **إجراءً واحدًا** واضحًا.

## 2. متى لا يُستخدم
- كبديل عن التحميل (استخدم هيكلًا عظميًا أثناء الجلب).
- لإخفاء خطأ حقيقي خلف رسالة محايدة (صِف الخطأ).
- لحشو رسوم أو نصوص تسويقية متحمّسة (ممنوع #6).

## 3. الحالات (أربعة أنواع، لكلٍّ نبرة)
- `first-use`: «لم تُضِف أي عميل بعد» + زر «أضف أول عميل».
- `no-results`: «لا نتائج تطابق هذه الفلاتر» + رابط «امسح الفلاتر».
- `error`: «تعذّر تحميل البيانات» + زر «أعد المحاولة».
- `no-permission`: «لا تملك صلاحية عرض هذا المحتوى» + «العودة» أو «تواصل مع المسؤول».
(لكلٍّ: عنوان + وصف سطر + إجراء واحد.)

## 4. المقاسات
- محتوى متمركز، حشو رأسي سخيّ `--space-3xl` (64) ليتنفّس.
- **العنوان:** الحجم البصري `--font-size-lg` `--font-weight-semibold` `--color-text-primary` — أمّا **العنصر الدلالي فعنوان** بمستوى يحدّده السياق (§بنية العنوان). الحجم ثابت مهما تغيّر المستوى.
- الوصف `--font-size-md` `--color-text-secondary`.
- **الأيقونة/الرسم (حسب السياق):**
  - **تطبيقات بيانات / صفحات إدارية:** أيقونة بسيطة أو visual بسيط فقط (`--size-icon-lg`، `--color-text-tertiary`) — **لا illustration زخرفي كبير، ولا أيقونة عامة بلا معنى**.
  - **تسويق / Landing:** يُسمح بـillustration إذا خدم الهوية أو وضّح المعنى (`aria-hidden` إن كان زخرفيًا؛ لا يحلّ محلّ العنوان/الإجراء ولا يؤخّر المحتوى).
- عرض كتلة النصّ ≤ ~`48ch` للقراءة.

## بنية العنوان والوصول (إلزامي — يمنع تكرار الخطأ)
> سبب الإضافة: تحديد **الحجم** وحده يدفع المنفّذ لاستخدام `<p>`، فتُكسر شجرة العناوين؛ وفي حالة تملأ الصفحة (خطأ/لا صلاحية) تبقى الشاشة **بلا أي عنوان** لقارئ الشاشة (خالف المبدأ 7). الدلالة منفصلة عن الحجم.

1. **عنصر عنوان دائمًا:** العنوان `<h1>`–`<h3>`، لا `<p>`/`<div>`/`<span>`. هذا ممنوع.
2. **المستوى حسب السياق (دلالة لا حجم):**
   - الحالة تستبدل **الصفحة كاملة** (خطأ تحميل، لا صلاحية، أول استخدام بلا رأس) ⇒ `<h1>` (هو عنوان الشاشة حينها).
   - الحالة **داخل قسم/كرت/جدول** تستبدل محتواه الفرعي ⇒ مستوى يلي عنوان حاويته (قسمها `<h2>` ⇒ عنوانها `<h3>`)، دون قفز مستويات.
3. **الحجم البصري يبقى `--font-size-lg`** أيًّا كان المستوى الدلالي — لا تخلط الدلالة بالبصر.
4. **الإعلان (للخطأ فقط):** حالة الخطأ التي تظهر ديناميكيًا تُغلَّف بـ`role="alert"` وتُحقَن في DOM عند الحدوث (لا أن تكون مخفيّة مسبقًا) لتُعلَن فور ظهورها. `first-use`/`no-results`/`no-permission` ليست تنبيهات، فلا `alert` لها.

## 5. قواعد النصوص
- العنوان يصف الوضع بإيجاز؛ الوصف يوجّه للخطوة.
- نبرة مهنية هادئة، بلا مبالغة/إيموجي ([ux-writing](../foundations/ux-writing.md)).
- إجراء بفعل مباشر يطابق نوع الحالة.

## 6. قواعد RTL
- المحتوى متمركز فلا انعكاس للتخطيط، لكن النصّ `text-align` يتبع المركز/البداية.
- أي أيقونة اتجاهية تُعكَس؛ الأرقام لاتينية ([rtl-guidelines](../foundations/rtl-guidelines.md)).

## 7. أمثلة صحيحة
- خطأ على مستوى الصفحة: `<h1>` «تعذّر تحميل البيانات» داخل `role="alert"` + «أعد المحاولة».
- «لا وصف بعد» داخل قسم عنوانه `<h2>` ⇒ عنوان الحالة `<h3>` + «أضف وصفًا».
- نتائج بحث صفرية: «لا نتائج لـ ‹...›» + «امسح الفلاتر».

## 8. أمثلة خاطئة
- **العنوان `<p>`/`<div>` بدل عنصر عنوان (يكسر شجرة العناوين) — الخطأ الأكثر شيوعًا.**
- صفحة خطأ/لا-صلاحية بلا أي `<h1>`.
- قفز المستويات (`<h2>` داخل قسم `<h2>`، أو `<h4>` مباشرة دون `<h3>`).
- «المكان فارغ هنا… ابدأ رحلتك! 🚀» (مبالغة/إيموجي، ممنوع #6).
- حالة بلا إجراء (طريق مسدود) / استخدامها أثناء التحميل بدل الهيكل العظمي.
- illustration زخرفي كبير في سياق المنتج (تطبيق بيانات/إداري) — مسموح في التسويق فقط إن خدم الهوية/المعنى.

## 9. علاقته بالتوكنز
- العنصر: عنوان `<h1>`–`<h3>` (دلالة) بحجم `--font-size-lg` (بصر) — فصْل صريح.
- النصّ: `--color-text-primary` (عنوان) · `--color-text-secondary` (وصف) · `--color-text-tertiary` (أيقونة).
- المسافات: `--space-3xl` (رأسي) · `--space-md`.
- الإجراء: توكنز [button](button.md).
