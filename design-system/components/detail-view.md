# عرض التفاصيل — Detail View / Description List

> **طبقة تركيب** (مثل [form](form.md))، لا مكوّن ذرّي: نمط عرض **عنصر واحد** (مستخدم/طلب/شركة/جلسة/فاتورة). القيم من [tokens.md](../foundations/tokens.md). **ممنوع تركيب `dl` يدويًا كل مرّة بلا هذا العقد** (ظهر في dogfood v2.1.0 — أُغلق هنا).

## الغرض
عرض خصائص كيان واحد بوضوح قابل للمسح: تسمية ↔ قيمة، مع رأس وحالة وإجراءات. **بديل عن الجدول** حين يكون العنصر واحدًا (presentation-by-content — [anti-bland §7](../rules/anti-bland-ui-rules.md)).

## متى يُستخدم
صفحة/درج تفاصيل عنصر واحد؛ ملخّص كيان أعلى شاشة؛ مراجعة قبل إجراء.

## متى لا يُستخدم
- مقارنة عدّة عناصر متجانسة → [table](table.md).
- إدخال/تحرير الحقول → [form](form.md) (هذا للعرض، لا الإدخال).
- محتوى سردي طويل → نصّ/أقسام عادية.

## التشريح (Anatomy)
```
[ page-header ]            ← عنوان الكيان + إجراء أساسي (تعديل…)
[ summary area ]           ← avatar/أيقونة + اسم + شارة حالة
[ key-value groups ]       ← dl (مجموعة أو أكثر بعناوين أقسام)
[ status / action area ]   ← الحالة + إجراءات ثانوية
[ metadata ]               ← منشئ/تواريخ (خافت)
[ related records ]        ← (اختياري) جداول/قوائم مرتبطة
```

## البنية (Structure — دلالية)
- `<dl>` للمجموعة · `<dt>` للتسمية · `<dd>` للقيمة. لكل مجموعة عنوان (`<h3>`/sec-label).
- التسمية `--color-text-secondary`، القيمة `--color-text-primary`.

## الأنواع (Variants)
- **simple details** — `dl` واحد.
- **grouped details** — عدّة `dl` بعناوين أقسام مفصولة بـ`--space-lg`/`--space-xl`.
- **summary + details** — رأس ملخّص (avatar+اسم+حالة) ثم المجموعات.
- **details with actions** — إجراء أساسي في الرأس + ثانوية قرب الحالة.
- **details with status** — شارة حالة في الملخّص.

## السلوك المستجيب (Responsive)
- **desktop:** عمودان للـ`dl` عند الحاجة (تسمية ثابتة ~`9rem` + قيمة مرنة)، أو شبكة مجموعات.
- **mobile:** **stacked** — التسمية فوق القيمة أو عمود واحد؛ لا عمودان مضغوطان.

## قواعد RTL
- التسميات والقيم `text-align: start`.
- البريد/المعرّفات/الروابط `<bdi>` أو `dir="ltr"` معزولة، مع بقاء التخطيط RTL.
- ترتيب الأعمدة (تسمية←قيمة) يتبع التدفّق المنطقي، لا `left/right`.

## الوصول
- `<dl>`/`<dt>`/`<dd>` دلالية (تربط التسمية بالقيمة لقارئ الشاشة).
- عنوان لكل مجموعة (`<h3>`) في التسلسل الصحيح تحت `<h1>` الكيان.
- تسميات واضحة لا مختصرات غامضة.

## القواعد البصرية (Visual rules)
- **لا تحشُر كل التفاصيل في كروت كثيرة بلا داعٍ** ([anti-bland §1](../rules/anti-bland-ui-rules.md)) — مجموعات داخل سطح واحد مفصولة بمسافة/خطّ، لا كرت لكل حقل.
- **لا تستخدم جدولًا لعرض key-value بسيط** لعنصر واحد (إساءة استخدام table).
- **لا مسافات عشوائية** — سلّم المسافات: تسمية↔قيمة `--space-sm`، صفّ↔صفّ `--space-sm`، مجموعة↔مجموعة `--space-lg`/`-xl`.
- توكنز فقط؛ الكثافة **comfortable** (تفصيل للقراءة لا جدول كثيف).

## Do / Don't
- **Do:** `dl` دلالي؛ مجموعات بعناوين؛ عمودان ديسكتوب/stacked موبايل؛ avatar+حالة في الملخّص.
- **Don't:** كرت لكل حقل؛ جدول لـkey-value مفرد؛ مسافات خارج السلّم؛ تكرار عنوان الكيان في كل مجموعة.

## ملاحظات Tailwind
- `dl`: `grid grid-cols-[label_value]` عبر شبكة منطقية (لا قيم arbitrary — استخدم توكن/utility)، أو `flex` صفوف على الموبايل.
- المسافات من السلّم؛ العناوين من السلّم الطباعي؛ راجع [component-tokens](../foundations/component-tokens.md).

## أمثلة
- **تفاصيل مستخدم:** رأس (avatar `lg` + اسم + شارة) → مجموعة «معلومات الحساب» (المعرّف/الدور/القسم/التواريخ) → مجموعة «الصلاحيات».
- **تفاصيل طلب تسجيل:** ملخّص (رقم + حالة) → بيانات المتقدّم → بيانات البرنامج → سجلّ مرتبط.
- **تفاصيل شركة:** avatar `rounded` + اسم → بيانات قانونية → جهات اتصال.

## علاقته بالتوكنز (عقد الربط)
| الجزء | التوكن الدلالي |
|---|---|
| التسمية `dt` | `--color-text-secondary` |
| القيمة `dd` | `--color-text-primary` |
| عنوان المجموعة | `--color-text-tertiary` (sec-label) + `--font-weight-bold` |
| السطح/الفاصل | `--color-bg-surface` · `--color-border-subtle` |
| الميتاداتا | `--color-text-tertiary` |
| المسافات | `--space-sm` (داخلي) · `--space-lg`/`-xl` (مجموعات) |
