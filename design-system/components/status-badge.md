# شارة الحالة — Status Badge

> القيم من [tokens.md](../foundations/tokens.md). تستخدم ألوان الحالات الوظيفية (غير القابلة للثيم).

## 1. متى يُستخدم
لعرض **حالة** كيان بإيجاز عالي القراءة داخل جدول/كرت/تفصيل: مكتمل، قيد المعالجة، مرفوض، مسوّدة.

## 2. متى لا يُستخدم
- كزر/رابط (الشارة عرض لا فعل) → [button](button.md).
- لعدّ أو كمّيات (استخدم رقمًا/عدّاد).
- لتلوين زخرفي بلا دلالة حالة (ممنوع #6).
- لا تُكثر ألوان الحالات بلا معنى ثابت لكل لون.

## 3. الحالات (تعيين دلالي ثابت)
- نجاح/مكتمل → `--color-success-*`
- قيد الانتظار/تحذير → `--color-warning-*`
- خطأ/مرفوض → `--color-danger-*`
- معلومة/قيد المعالجة → `--color-info-*`
- محايد/مسوّدة → `--neutral` (bg-subtle + text-secondary)
كل شارة: خلفية `-bg` + نصّ `-fg` + حدّ `-border` (أو نقطة لون + نصّ).

## 4. المقاسات
- ارتفاع مدمج، حشو `--space-2xs` رأسي / `--space-xs` أفقي، زاوية `--radius-sm` (أو `--radius-full` للنمط الحبّة).
- النصّ `--font-size-xs`/`-sm` `--font-weight-medium`.
- نقطة لون اختيارية `8px` بفجوة `--space-2xs`.

## 5. قواعد النصوص
- كلمة/كلمتان تصف الحالة: «مكتمل»، «قيد المراجعة».
- مصطلح حالة واحد ثابت لكل قيمة عبر التطبيق (المعجم، [ux-writing](../foundations/ux-writing.md)).
- لا جُمل ولا أرقام داخل الشارة.

## 6. قواعد RTL
- النقطة في `inline-start` والنصّ يليها، بخصائص منطقية.
- الشارة غير اتجاهية فلا تُعكَس؛ نصّها عربي يتبع RTL ([rtl-guidelines](../foundations/rtl-guidelines.md)).

## 7. أمثلة صحيحة
- «مكتمل» بأخضر (`success`) في عمود حالة الطلب.
- «مرفوض» بأحمر (`danger`) + نقطة لون.
- «مسوّدة» محايدة رمادية.

## 8. أمثلة خاطئة
- شارة قابلة للنقر تُطلق فعلًا (اخلطها مع زر).
- الاعتماد على اللون وحده بلا نصّ (يفشل لعمى الألوان والوصول).
- ألوان حالة خارج التوكنز أو معانٍ متضاربة (أخضر لمرفوض).
- شارة بعملة/رقم (ليست عدّادًا).

## 9. علاقته بالتوكنز
- الحالات: `--color-{success|warning|danger|info}-fg`/`-bg`/`-border`.
- المحايد: `--color-bg-subtle` · `--color-text-secondary` · `--color-border-subtle`.
- القياس: `--space-2xs`/`-xs` · `--radius-sm`/`--radius-full`.
- الخط: `--font-size-xs`/`-sm` · `--font-weight-medium`.
