# التخطيط والشبكة — Layout & Grid

> يحكم بنية الصفحة على مستوى عالٍ. القيم من [tokens.md](tokens.md)؛ المسافات من [spacing.md](spacing.md). كل شيء هنا **منطقي الاتجاه** (يبدأ من اليمين في RTL).

---

## نقاط الكسر — Breakpoints

```
sm:  640px
md:  768px
lg:  1024px
xl:  1280px
2xl: 1536px
```
**السبب:** عناقيد الأجهزة الشائعة — هاتف (<640)، لوحي (768)، لوحي أفقي/حاسوب صغير (1024)، حاسوب (1280)، شاشة واسعة (1536). **لأن المنتج تطبيق بيانات**، نصمّم **من الأعلى للأسفل (desktop-first)** لأن الاستخدام الأساسي على الحاسوب، مع ضمان عمل الجوال.

القاعدة: استعلامات الوسائط بالخصائص المنطقية حيثما أمكن، والقياس بالحدّ الأدنى المتّسق (`min-width`).

---

## عرض المحتوى

| السياق | العرض الأقصى | السبب |
|---|---|---|
| هيكل التطبيق (shell) | `1536px` ثم يتمركز | يمنع تمدّد الجداول لعرض غير مقروء على الشاشات العريضة |
| منطقة المحتوى داخل الشريط الجانبي | مرنة (fluid) | تملأ المتاح بعد الشريط |
| نص قرائي (مقالات، نصوص طويلة) | `65–75ch` | عرض سطر مريح للقراءة؛ أطول يُتعب العين |
| نموذج بعمود واحد | `480–640px` | يحفظ القرب بين التسمية والحقل |

## هيكل التطبيق — App Shell

النمط القياسي لتطبيق البيانات:

```
┌─────────────────────────── شريط علوي (topbar) ──────────────────────────┐
│  [الشعار]              [بحث]                    [إشعارات] [حساب]          │
├──────────────┬───────────────────────────────────────────────────────────┤
│              │                                                            │
│   الشريط     │                  منطقة المحتوى                            │
│   الجانبي    │                                                            │
│  (تنقّل)      │   عنوان الصفحة + الإجراء الأساسي                          │
│              │   ───────────────────────────────                         │
│  ابتداءً من   │   المحتوى                                                  │
│  اليمين      │                                                            │
└──────────────┴───────────────────────────────────────────────────────────┘
```

**في RTL:** الشريط الجانبي على **اليمين** (جهة البداية `inline-start`)، ومنطقة المحتوى يسارها. هذا انعكاس تلقائي عند استخدام الخصائص المنطقية — لا يُبنى التخطيط بـ`left`/`right`.

- الشريط الجانبي: عرض ثابت (مثل `260px`)، قابل للطي إلى أيقونات (`64px`).
- الشريط العلوي: ارتفاع `--size-control-lg` تقريبًا + حشو، لاصق (`--z-sticky`).
- منطقة المحتوى: حشو `--space-lg` إلى `--space-xl`، تمرير مستقل.

## الشبكة — Grid

شبكة **12 عمودًا** للمناطق المرنة.

**السبب في 12:** تقبل القسمة على 2 و3 و4 و6، فتدعم تخطيطات (نصفين، أثلاث، أرباع) دون كسور — أكثر مرونة من 10 أو 16.

```css
.grid {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: var(--space-md);            /* أو --space-lg للمناطق الفسيحة */
}
```

- الفجوة (gutter): `--space-md` (16) افتراضيًا، `--space-lg` (24) للوحات المعلومات الفسيحة.
- **تدفّق الأعمدة في RTL من اليمين لليسار** تلقائيًا مع `dir="rtl"` — لا تعكس ترتيب عناصر DOM يدويًا.
- على الجوال (`<md`): الانهيار إلى عمود واحد؛ لا تُحشر 4 بطاقات في صف ضيق.

### أنماط شائعة معتمدة
- **لوحة مؤشرات:** صف بطاقات KPI (2–4)، ثم رسم/جدول بعرض كامل.
- **قائمة + تفصيل:** قائمة على البداية (يمين)، تفصيل على البقية.
- **نموذج:** عمود واحد متمركز؛ لا تُوزّع حقول مترابطة على عمودين إلا لو كانا متلازمين فعلًا (مدينة/رمز بريدي).

---

## الالتصاق والتمرير

- رأس الجدول لاصق (`position: sticky; --z-sticky`) في الجداول الطويلة.
- شريط الإجراءات الأساسي قد يلتصق أسفل المودال/النموذج الطويل.
- منطقة المحتوى تُمرَّر مستقلةً عن الشريط الجانبي والعلوي (تبقيان ثابتين).

---

## قواعد التجاوب

1. لا يكسر التجاوب **التسلسل المعلوماتي**: ما هو أهم على الحاسوب يبقى أهم على الجوال.
2. الجداول العريضة على الجوال: تمرير أفقي داخل الجدول، أو تحوّل لبطاقات صف-لكل-سجل — لا تُقصّ أعمدة بصمت (انظر [table](../components/table.md)).
3. هدف اللمس ≥ 44px على كل المقاسات.
4. لا تُخفِ إجراءً أساسيًا خلف قائمة على الجوال؛ الثانوية فقط تُطوى.
