# نظام الألوان الذكي — Smart Color System

> **القرار:** أي لون علامة يدخل النظام يُحوَّل إلى **نظام كامل** (ramp + semantic + ربط مكوّنات)، لا مجرد تبديل `primary`.
> **السبب:** تغيير لون واحد يجب أن ينعكس على كل النظام بشكل متوقّع ومحقَّق التباين — لا قيم عشوائية متناثرة.
> **القيد:** بلا أدوات بناء (لا npm/CLI). الاشتقاق **بروتوكول ينفّذه الوكيل** بقواعد محدّدة، ويُفحَص بـQA. يُكمّل [brand-foundation](brand-foundation.md) (طبقات الثيم) و[tokens.css](../tokens.css) (المصدر التنفيذي).

---

## 1. المعمارية اللونية (تير الـramp فوق الطبقات الثلاث)

```
البذرة (لون علامة واحد)
   ↓ اشتقاق (§3)
Brand Ramp:  --brand-50 … --brand-950   ← أوّليات العلامة (11 درجة)
   ↓ ربط (§4)
Semantic:    --color-primary / -hover / -active / -subtle / -foreground · link · focus-ring
   ↓ يستهلكها
Components:  عقد الربط لكل spec (الطبقة 3) — تربط بالـsemantic فقط
```

القاعدة الثابتة (من [brand-foundation](brand-foundation.md)): **المكوّنات لا تلمس الـramp ولا الخام — تستهلك الدلالي فقط.** فالتتالي تلقائي: بذرة جديدة → ramp جديد → semantic جديد → كل المكوّنات تتحدّث بلا لمسها.

**ما لا يتغيّر مع العلامة (وظيفي):** المقياس الرمادي `--neutral-1…12`، ألوان الحالات (success/warning/danger/info)، سلّم المسافات/الطباعة/الكسر. العلامة تغيّر **الـbrand ramp فقط** (وما يشتقّ منه).

---

## 2. الإدخال (أي صيغة بذرة)

تُقبل البذرة بأي شكل: `#1277A1` · «بيج» · «أزرق حكومي رسمي» · اسم لون. الوكيل يحوّلها إلى hex واحدة (البذرة)، ثم يطبّق §3. مثال البيج/الفاتح يدخل **فرع البراند الفاتح** (§5).

---

## 3. الاشتقاق — البروتوكول (ينفّذه الوكيل)

**خطوة 1 — حوّل البذرة إلى OKLCH** `(L, C, H)`. (OKLCH لأن الإضاءة فيه إدراكية مستوية — يمنع الـramp «الموحل» أو «النيون».)

**خطوة 2 — حدّد دور البذرة من إضاءتها L:**
| L البذرة | الدور | المعالجة |
|---|---|---|
| 0.45 – 0.72 | **لون فعل** (الشائع) | البذرة تثبّت قرب 500/600؛ ولّد كامل الـramp |
| > 0.80 | **علامة فاتحة** (بيج/ليموني/باستيل) | فرع §5: البذرة تصبح **سطح** لا فعل |
| < 0.30 | **علامة داكنة** | البذرة قرب 800/900؛ الفعل منها، الـforeground أبيض |

**خطوة 3 — ولّد 11 درجة بإضاءة ثابتة (L مستهدَفة) ومنحنى chroma:**

| الدرجة | L (OKLCH) | معامل C (× C البذرة) |
|---|---|---|
| 50  | 0.971 | 0.16 |
| 100 | 0.936 | 0.30 |
| 200 | 0.885 | 0.46 |
| 300 | 0.808 | 0.66 |
| 400 | 0.704 | 0.86 |
| 500 | 0.616 | 1.00 (الذروة) |
| 600 | 0.540 | 0.96 |
| 700 | 0.464 | 0.88 |
| 800 | 0.399 | 0.76 |
| 900 | 0.348 | 0.62 |
| 950 | 0.258 | 0.44 |

- **H ثابت** عبر كل الدرجات (لا انزياح صبغة).
- **C = C البذرة × المعامل**، ثم **قُصّ على نطاق sRGB** (إن خرجت درجة عن النطاق، اخفض C حتى تدخل — لا تغيّر L).
- **تثبيت البذرة (anchor):** أسنِد لون البذرة لأقرب درجة بإضاءتها (عادة 500 أو 600) حتى يبقى لون العلامة الأصلي حاضرًا في الـramp.

**خطوة 4 — حوّل كل درجة من OKLCH إلى sRGB hex.** الناتج: 11 قيمة تُكتب في `--brand-50…950` (§6).

---

## 4. الربط بالـSemantic + حُرّاس التباين

الربط الافتراضي (واجهة فاتحة، خلفية `--color-bg-base`):

| التوكن الدلالي | الدرجة الافتراضية | حارس التباين (إلزامي) |
|---|---|---|
| `--color-primary` | **brand-600** | ≥ 4.5:1 مع `--color-primary-foreground` |
| `--color-primary-hover` | brand-700 | — |
| `--color-primary-active` | brand-800 | — |
| `--color-primary-subtle` | brand-50 | نصّ فوقه ≥ 4.5:1 (عادة brand-700/800) |
| `--color-primary-foreground` | أبيض أو `--neutral-12` | يُختار أيّهما يحقّق ≥ 4.5:1 فوق primary |
| `--color-text-link` | brand-700 | ≥ 4.5:1 فوق bg-base |
| `--color-focus-ring` | brand-600 | ≥ 3:1 فوق bg-base |
| `--color-accent*` (أسماء موجودة) | تُربَط بالأعلى للتوافق الخلفي | — |

**قاعدة الحارس (إن فشل تباين):** تدرّج للدرجة الأغمق التالية حتى يمرّ (primary 600→700→800؛ link 700→800). **لا تُقبل قيمة تكسر AA.** هذا يطبّق فصل link/focus عن accent الموجود في [brand-foundation](brand-foundation.md) §شروط(6) آليًّا.

**الأسطح والنصوص والحدود تبقى محايدة افتراضيًا.** (اختياري متقدّم: سطح مُلمَّح بـbrand-50 — يُفعَّل صراحةً فقط، ولا يُطبَّق تلقائيًا حفاظًا على مبدأ «الثيم لا يلمس الرمادي جماليًّا».)

---

## 5. فرع العلامة الفاتحة (L > 0.80)

البذرة الفاتحة **لا تصلح لونَ فعل** (تكسر التباين كتعبئة زر). المعالجة (مطابقة [theme-moonli](../themes/theme-moonli.md)):
- `--color-brand` (سطح/تظليل علامة) = البذرة (≈ brand-200) · `--color-on-brand` = brand-950 (داكن، ≥ 4.5:1).
- **لون الفعل** `--color-primary` = **brand-900/950** (أو `--neutral-12`) — داكن عالي التباين.
- `--color-text-link` و`--color-focus-ring` = درجة داكنة محقَّقة التباين، لا البذرة الفاتحة.
بهذا تظهر العلامة الفاتحة كهوية (أسطح/حدود) دون أن تنافس الفعل أو تكسر الوصول.

---

## 6. كيف يُكتب (tokens.css / الثيم)

البذرة + الـramp المشتق يُكتبان كـ**ثيم** يُعيد تعريف `--brand-*` فقط؛ والـaccent/semantic مربوطة بالـramp في [tokens.css](../tokens.css) فتتحدّث تلقائيًا:

```css
:root[data-theme="brand-x"]{
  /* الـramp المشتق (§3) — 11 درجة */
  --brand-50:#…; --brand-100:#…; … --brand-600:#…; … --brand-950:#…;
  /* عند الحاجة فقط: تجاوز ربط افتراضي (مثلًا فرع فاتح §5) */
  --color-primary-foreground:#FFFFFF;
}
```
الافتراضي المحايد في tokens.css هو **ramp نائب** (أزرق `#3B5BDB` عند 600) موسوم نائبًا — يُستبدل عند اعتماد علامة.

## 7. مثال محلول — البذرة `#1277A1` (فيروزي)

OKLCH ≈ `L 0.47 · C 0.09 · H 221`. دور = **لون فعل** (L في النطاق). الـramp المشتق (§3):

| 50 | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900 | 950 |
|---|---|---|---|---|---|---|---|---|---|---|
| `#ECF6FA` | `#D3EAF2` | `#A9D6E6` | `#74BBD4` | `#3E9CBF` | `#1A83A9` | `#1277A1` | `#0F6285` | `#0D506C` | `#103F54` | `#0A2A39` |

الربط (§4) + فحص التباين:
- `--color-primary` = brand-600 `#1277A1` · مع أبيض ≈ **4.6:1 ✓** (يمرّ AA؛ لو فشل لانتقلنا 700).
- `--color-primary-hover` = 700 `#0F6285` · `-active` = 800 `#0D506C` · `-subtle` = 50 `#ECF6FA`.
- `--color-text-link` = 700 `#0F6285` على الأبيض ≈ **6.8:1 ✓** · `--color-focus-ring` = 600 ≈ **3.2:1 ✓**.
- `--color-primary-foreground` = أبيض (يحقّق ≥4.5 فوق 600).
النتيجة: تغيير البذرة وحدها حدّث الأزرار/الروابط/الحلقات/التلميحات — بلا لمس أي مكوّن.

---

## 8. تنفيذ الوكيل + QA (إلزامي)

عند إعطاء المستخدم بذرة:
1. استنتج الدور (§3 خطوة 2) — إن فاتحة، فرع §5.
2. ولّد الـramp (§3) واكتبه ثيمًا (§6).
3. اربط الـsemantic (§4) وطبّق حُرّاس التباين (تدرّج حتى المرور).
4. **QA لوني (يُسجَّل):** primary/foreground ≥4.5 · link ≥4.5 · focus-ring ≥3 · primary كحدّ/أيقونة ≥3 · subtle/نصّه ≥4.5 · لون فعل واحد (لا ثانٍ منافس) · لم تُمسّ الحالات/الرمادي.
5. أي فحص يفشل ولا يُصلَح بالتدرّج → **بوّابة Design System لا تمرّ** (التنفيذ غير صالح).

> الاشتقاق تقريب إدراكي ينفّذه الوكيل، لا حساب آلي — لذا **فحص التباين في الخطوة 4 إلزامي** ويحكم القبول، لا مجرد توليد القيم.
